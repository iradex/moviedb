Documentation - MovieDB API

Description Endpoint Method Type Query Parameters Request Data Format Response Data Format
Returns list of all movies. /movies GET - - A JSON Object with data about all the movies
Returns information about a single movie /movies/[Title] GET [Title] is the title of the movie for which the details will be responded. - A JSON Object containing details about the movie that was requested via the endpoint. Example:
{ title: "Forrest Gump", actor: "Tom Hanks", length: 186 }
Returns information about a certain genre. /movies/genres/[genre] GET [Genre] is the genre of the movies for which the list will be generated. - A description of a certain genre, example: "Drama is a genre of narrative fiction (or semi-fiction) intended to be more serious than humorous in tone. "
Returns information about a certain director /directors/[director’s name] GET [Director’s name] is the name of the director for which the information will be displayed. - A JSON Object containing the information about the particular director. Example: { "Name": "Steven Spielberg", "Bio": "One of the most influential personalities in the history of cinema, Steven Spielberg is Hollywoods best known director and one of the wealthiest filmmakers in the world.", "Birth": "1946" }
Creates a new user /users POST - A JSON Object holding the name of the user as well as the password. Example:
{ username: „MovieFriend“, password: „12345“, mail: „moviefriend@gmail.com", birthday: "01-14-1991", favoriteMovies: [] }		 A message that the user was successfully created
Updates user data /users/[username] PUT [Username] is the username of the current user… A JSON Object holding the name of the user as well as his key data. Example:
{ username: „MovieFriend“, password: „123456“, mail: „moviefriend@gmail.com", birthday: "01-14-1991", favoriteMovies: [] } 		A message that the user data was updated successfully.
Lets users add a list to their personal movie list /users/[username]/favorites/[movie id] POST [Username] is the username of the current user, while [movie id] is the ID of the movie to be added to the user's favorites. - A message showing that the title was added succesfully.
Lets users delete a movie from their favorites list /users/[username]/favorites/[movie id] DELETE [Username] is the username of the current user. [movie id] is the id of the movie that should be deleted. - A message showing that the title was successfully removed from the favorites list.
Lets users delete their account. /users/[username] DELETE [Username] is the username of the current user… - A message showing that the user was successfully deleted.
